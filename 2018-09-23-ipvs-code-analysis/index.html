ipvs与keepalived源码分析

Posted on 2018-09-23 | In drafts |

LVS的负载均衡是通过linux内核的IPVS模块实现的。IPVS模块是LVS集群的核心模块，运行在负载均衡节点上。keepalived是集群管理中保证集群高可用的服务软件，负责LVS服务器的高可用。

本文主要分析alibaba开源的alibaba/LVS

ipvs架构与初始化

ipvs在linux内核中的源码文件如下，路径在 net/netfilter/ipvs 中。

ipvs模块是以内核插件的形式存在。ipvs的初始化函数ip_vs_init。初始化函数中，会进行ipvs与netlink接口相关的初始化，ipvs协议相关的初始化，app相关的初始化以及连接相关的初始化。

ip_vs_control_init函数注册了与netlink的sockopt接口。
ip_vs_protocol_init函数注册了tcp、udp、ah、esp协议的相关接口。
ip_vs_app_init函数注册了app的/proc/net相关接口。
ip_vs_conn_initipvs连接相关的初始化，哈希表的per_cpu变量初始化，连接的哈希表锁初始化，ipvs连接cache的初始化等。

netfilter与hook函数

linux网络协议栈的简化图：

netfilter的5个hook点：

  • PRE_ROUTING
  • LOCAL_IN
  • FORWARD
  • LOCAL_OUT
  • POST_ROUTING

ipvs模块基于netfilter模块的hook点来截取报文。
ip_vs_in函数挂接在netfilter的PRE_ROUTING点，ip_vs_out函数挂接在netfilter的FORWARD点。

ipvs模块初始化时，向netfilter注册hook点：

ip_vs_in函数是数据包经过LVS时的总入口，从client到LVS或者RS到LVS的数据包都经过ip_vs_in函数的处理。
ip_vs_in函数首先会检查报文目的是否本主机，报文协议是否支持。然后根据协议获取初始化时注册的protocol结构指针，由协议结构内挂接的连接获取函数来获取连接，如果连接不存在则新建。
如果连接存在，根据报文的入方向（client->lvs）或出方向(rs->lvs)，将报文处理，增加信息统计并发送等。
